# 📊 Telecom Customer Churn Analysis

## 📌 Project Overview
This project analyzes customer churn behavior in a telecom company using SQL and Power BI. 

The goal is to identify key drivers of churn, segment customers based on risk, and provide actionable business recommendations to improve retention.

---

## 🎯 Objectives
- Analyze churn rate and customer behavior
- Identify key factors influencing churn
- Build a churn risk segmentation model using SQL
- Create an interactive Power BI dashboard for business insights

---

## 🛠 Tools Used
- SQL (data cleaning, analysis, feature engineering)
- Power BI (dashboard & visualization)
- Excel (optional validation)

---

## 🧹 Data Cleaning
- Checked for duplicates and missing values
- Validated data types and ranges
- Ensured dataset consistency before analysis

---

## 📊 Key Analysis

### 🔹 Churn Rate
- Calculated overall churn rate across all customers

### 🔹 Customer Segmentation
- Analyzed churn by:
  - Contract type
  - Tenure group
  - Monthly charges
  - Internet service

---

## 🧠 Churn Risk Model (SQL-Based)

Customers were segmented into risk groups:

- 🔴 High Risk:
  - Month-to-month contracts
  - High monthly charges
  - Low tenure

- 🟡 Medium Risk:
  - Partial risk factors

- 🟢 Low Risk:
  - Long-term contracts or loyal customers

## 🔍 Key Insights
- Month-to-month customers have the highest churn rate
- New customers (low tenure) are most likely to leave
- Higher monthly charges are associated with increased churn
- High-risk customers represent a small but critical segment

---

## 💡 Business Recommendations
- Offer incentives for long-term contracts
- Improve onboarding experience for new customers
- Target high-risk customers with retention campaigns
- Review pricing strategy for high-charge segments
